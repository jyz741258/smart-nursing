# 智慧养老系统 - 大数据分析功能说明

## 📊 功能概述

新增的大数据分析功能提供了全面的健康数据智能分析，包括：

### 1. 健康指标分析
- **统计分析**：计算心率、血压、血糖、体温等指标的最大值、最小值、平均值、中位数、标准差
- **异常检测**：自动识别超出正常范围的指标
- **趋势分析**：对比近期与历史数据，发现健康趋势变化

### 2. 趋势预测
- **简单线性回归**：基于历史数据预测未来健康趋势
- **预测周期**：可预测未来7天的指标变化
- **置信度评估**：显示预测结果的可靠性

### 3. 风险评估
- **多维度评估**：从多个健康指标综合评估风险等级
- **风险分级**：高风险、中风险、低风险
- **风险因素识别**：明确指出导致风险的具体指标

### 4. 活动模式分析
- **睡眠分析**：统计睡眠时长和睡眠质量
- **步数统计**：记录每日活动量
- **活动等级划分**：高活动、中等活动、低活动

### 5. 异常告警
- **实时监测**：自动检测异常指标
- **告警分级**：紧急告警（critical）和一般警告（warning）
- **健康建议**：针对异常情况提供改善建议

### 6. 智能洞察
- **数据完整度**：评估健康数据的记录情况
- **测量频率**：分析测量频次是否足够
- **综合评分**：生成健康评分和改善建议

## 🔌 后端API接口

所有接口均需要登录认证，位于 `/api/bigdata/` 前缀下：

| 接口 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/health-analysis` | GET | elder_id, days | 获取综合健康分析 |
| `/elder-health-report` | GET | elder_id, days | 获取老人健康报告 |
| `/health-prediction` | GET | elder_id, metric_type, days, predict_days | 健康趋势预测 |
| `/nursing-efficiency` | GET | days | 护理效率分析 |
| `/care-plan-analysis` | GET | - | 护理计划完成度分析 |
| `/risk-assessment` | GET | elder_id, days | 老人健康风险评估 |
| `/activity-patterns` | GET | elder_id, days | 活动模式分析 |
| `/comparison` | GET | elder_ids, metric_type, days | 多老人数据对比 |
| `/anomaly-alerts` | GET | elder_id, days | 异常告警列表 |
| `/health-insights` | GET | elder_id, days | 健康数据洞察 |

## 📱 前端页面

访问路径：`/bigdata` 或通过侧边栏"大数据分析"菜单

### 主要功能模块

1. **健康评分卡片**：直观展示整体健康状况
2. **智能洞察面板**：AI生成的数据质量评估
3. **健康指标图表**：交互式图表展示各项指标
4. **趋势预测图**：可视化预测未来趋势
5. **风险评估**：多维度风险等级展示
6. **活动模式**：睡眠和步数趋势分析
7. **异常告警**：实时告警信息推送

## 🚀 使用方法

### 1. 启动后端服务

```bash
cd backend
python -m flask run --host=0.0.0.0 --port=5000
```

### 2. 启动前端服务

```bash
cd frontend
npm run dev
```

### 3. 访问大数据分析页面

1. 使用管理员账号登录
2. 在左侧菜单点击"大数据分析"
3. 选择要分析的老人（可选）
4. 设置时间范围（7天/30天/90天）
5. 点击"刷新数据"查看分析结果

## 📊 数据来源

分析数据来源于以下模块：

- **健康指标记录** (`health_metrics` 表)
- **护理记录** (`nursing_records` 表)
- **护理任务** (`care_tasks` 表)
- **护理计划** (`care_plans` 表)

确保这些表中有足够的测试数据才能看到完整的分析结果。

## 🎯 注意事项

1. **数据量要求**：某些分析功能（如趋势预测）需要至少3条数据记录才能运行
2. **时间范围**：建议选择30天或90天以获得更准确的分析结果
3. **权限控制**：目前所有登录用户都可以访问此功能，可以根据需要添加角色权限限制
4. **性能考虑**：大量数据分析可能需要等待几秒钟，请耐心等待
