## 1. 简介

### 1.1 目的

“坤坤健康·智能老年护理系统”软件测试计划文档有助于实现以下目标：

a. 确定智能老年护理系统的信息和应测试的软件构件。  
b. 确定测试需求。  
c. 确定测试策略，并对这些策略加以说明。  
d. 确定所需的资源，并对测试的工作量进行估计。  
e. 列出项目测试的可交付工件。

### 1.2 背景

坤坤健康·智能老年护理系统是用于对老年护理的数字化统筹管理，对护理过程中产生的各种数据进行分析，为加快发展老年医疗护理服务，探索创新多元化老年医疗护理服务模式，积累老年医疗护理服务的有益经验。

### 1.3 范围

a. 对于本项目的测试工作为系统测试阶段，阶段的测试活动有计划测试、设计测试、执行测试和评估测试四个活动。  
b. 本测试计划仅对项目开发的软件提供测试，不考虑对第三方提供的软件（如第三方的组件或构件）进行测试。但需测试第三方提供的软件与开发的软件之间的接口。  
c. 本测试计划适用于“坤坤健康·智能老年护理系统”项目。本文档将供给项目经理及项目开发各组使用，包括测试组、开发组。

### 1.4 定义

**缺陷级别定义：**

- **一级：** 不能完全满足系统要求，基本功能未完全实现；或者危及人身安全。  
- **二级：** 严重影响系统要求或基本功能的实现，且没有更正办法（重新安装或重新启动该软件不属于更正办法）。  
- **三级：** 严重影响系统要求或基本功能的实现，但存在合理的更正办法（重新安装或重新启动该软件不属于更正办法）。  
- **四级：** 使操作者不方便或遇到麻烦，但它不影响执行工作功能或重要功能。  
- **五级：** 其他缺陷。

## 2. 测试需求

**备注：优先级：1-高，2-中，3-低**

### 2.1 功能测试需求

| 测试需求 | 测试需求简介 | 优先级 |
| --- | --- | --- |
| 用户注册登录 | 用户通过账户、密码/验证码方式注册或登录系统。 | 2 |
| 用户订单管理 | 用户（包括管理员）通过用户界面进行创建、更改、查看、（一定条件下）取消订单。 | 1 |
| 用户数据提交 | 用户通过用户界面提交在服务过程中产生的数据，包括满意度等。 | 1 |
| 护工资源管理 | 护工管理包括编辑护工信息、护工账号设置、删除护工、查看护工资质、多条件查询等操作。 | 1 |
| 护理服务管理 | 护理服务管理包括（管理员）新增护理服务、编辑护理服务信息、删除护理服务、查看护理服务详情、导出报表、多条件查询等操作。 | 1 |
| 新闻公告管理 | 新闻公告管理包括新增新闻公告、编辑新闻公告信息、审批新闻公告、删除新闻公告、查看新闻公告详情、导出Excel、多条件查询等操作。 | 2 |
| 平台人员管理 | 平台人员管理包含对护工、老人和家属的人员信息进行管理，功能统一包括新增平台人员、编辑平台人员信息、启停平台人员、删除平台人员、查看平台人员详情、多条件查询等操作。 | 1 |
| 退出登录 | 退出登录是后台管理员登陆后并退出的操作。 | 3 |

## 3. 测试策略

### 3.1 测试类型

本次测试为系统测试。

#### 3.1.1 功能测试策略

| 项目 | 内容 |
| --- | --- |
| **测试目标** | 所有功能测试需求项的功能实现。 |
| **测试方法和技术** | 按照测试需求、通过准则、测试用例，采用黑盒法测试，核实以下内容：<br>- 在使用合法数据时得到正确的结果。<br>- 在使用非法数据时显示相应的缺陷信息或警告信息。<br>- 各业务规则都得到了正确的应用。 |
| **完成标准** | - 所计划的测试已全部执行。<br>- 所发现的缺陷修复率达到通过准则要求。<br>- 提供《测试记录》。 |
| **需考虑的特殊事项** |  |

### 3.2 测试工具

| 名称 | 工具 | 版本 |
| --- | --- | --- |
| 用于功能性测试的工具 | Pytest, Cypress | 9.0 |
| 性能测试工具 | LoadRunner | 11.0 |

### 3.3 测试通过准则

- 实行了所有的测试策略并达到完成标准。  
- 测试结束后，开发组对实现有误的测试需求项的修改达到如下程度：  
  - 一、二级缺陷的修复率达到100%  
  - 三、四级缺陷的修复率达到80%以上  
  - 五级缺陷的修复率达到60%以上  
- 《需求规格说明书》、《用户手册》、《操作手册》和编码实现一致。

## 6. 附录 A：项目任务

以下是一些与测试有关的任务：

### 制定测试计划

- 确定测试需求  
- 评估风险  
- 制定测试策略  
- 确定测试资源  
- 创建时间表  
- 生成测试计划  

### 设计测试

- 准备工作量分析文档  
- 确定并说明测试用例  
- 确定测试过程，并建立测试过程的结构  
- 复审和评估测试覆盖  

### 执行测试

- 执行测试过程  
- 评估测试的执行情况  
- 恢复暂停的测试  
- 核实结果  
- 调查意外结果  
- 记录缺陷  

### 对测试进行评估

- 评估测试用例覆盖  
- 评估代码覆盖  
- 分析缺陷  
- 确定是否达到了测试完成标准与成功标准
